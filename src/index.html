what is weave?

Weave is a framework that lives inside your AI coding tool.

It doesn't replace your tools — it makes them smarter.

Through a natural conversation, Weave learns who you are,

what you're building, and how you like to work.

Then it generates a fully personalized workflow — agents, teams, rules, and skills —

tailored specifically to you and your project.

No config files to write.

No templates to copy.

Just a conversation.

features

├── onboarding.md# AI interviews you, builds your workflow
A natural conversation that understands your role, your project, and your preferences. No forms, no config — just talk. Weave analyzes your codebase too, so it knows what you're working with.
├── agents.md# Specialized AI agents for your project
Custom agents tailored to your stack — a reviewer that knows your ORM patterns, a planner that understands your architecture, a tester that matches your testing style. Each agent has one job and does it well.
├── teams.md# Agents that work together
Groups of agents with defined workflows. Your feature-dev team: Planner breaks it down, Coder implements, Reviewer checks quality, Tester validates. With clear handoffs and rejection handling.
├── evolve.md# Your setup improves while you work
Hooks track how you work in the background. When patterns emerge — new frameworks, repeated tasks, shifting focus — Weave proposes targeted improvements. Nothing changes without your approval.
├── sync.md# One workflow, every platform
Your workflow works across Claude Code, OpenClaw, Cursor, and Codex. Set it up once, sync everywhere. Each platform gets native-format files — no compatibility hacks.
└── uninstall.md# Clean removal, zero residue
Don't like it? npx weave-ai uninstall removes everything — every file, every config, every hook. Leaves your project exactly as it was. No lock-in, no leftovers.

Why Weave?

AI coding tools are powerful, but they don't know you. They don't know your project's architecture, your coding conventions, or how you like to break down tasks. So you end up repeating yourself — explaining context, correcting habits, re-establishing rules every session.

Weave fixes this. It's a framework that lives inside your AI tool and gives it persistent, personalized knowledge about:

  • Who you are — your role, experience, and how you like to work
  • What you're building — your tech stack, architecture, and project goals
  • How you want AI to behave — your rules, conventions, and preferences

Instead of configuring this manually, Weave generates everything through a conversation. And it keeps improving as your project evolves.

What makes Weave different

  • No templates — Everything is generated specifically for you and your project
  • Lives inside your tool — Not a separate CLI or service. It runs as native skills/commands within Claude Code, Cursor, Codex, or OpenClaw
  • Conversation-based — The onboarding is a natural dialog, not a form
  • Auto-evolving — Hooks track your patterns and suggest improvements
  • Multi-platform — One source of truth, synced to all your AI tools natively
  • Zero lock-in — Uninstall removes everything cleanly

Installation

Weave installs via npx — no global packages needed:

$ npx weave-ai

The installer will:

  • Show the Weave banner
  • Auto-detect which AI tools are present in your project (checks for .claude/, .cursor/, skills/, .codex/)
  • Let you choose which platforms to install for (comma-separated numbers)
  • Install platform-native files for each selected platform
  • Create the .weave/ state directory

Onboarding

After installation, open your AI tool in the project and run:

/weave:onboarding

This starts a natural conversation (~10 exchanges) in 3 phases:

Phase 1: The Person

Weave learns about you — your role, experience, primary tech stack, how you prefer to work with AI (plan first vs. dive in, terse vs. detailed responses), and what frustrates you about your current workflow.

Phase 2: The Project

Weave analyzes your codebase (package.json, directory structure, frameworks) and asks about your vision, goals, team, and milestones. It doesn't just rely on your answers — it reads the code too.

Phase 3: Workflow Proposal

Based on everything learned, Weave proposes a specific workflow:

  • Which agents to create and why
  • Which teams to assemble with what workflow
  • Which rules to enforce
  • Which custom skills to add

You review, adjust, and approve. Then Weave generates all the files.

What gets generated

  • .weave/config.json — Your profile and project context
  • .weave/agents/*.md — Agent definitions
  • .weave/teams/*.json — Team configurations
  • .weave/rules/*.md — Project rules
  • .weave/skills/*.md — Custom skills
  • Platform-specific files (CLAUDE.md, SOUL.md, etc.)

Agents

Agents are specialized AI personas with a single clear job. Each agent has:

  • Role — One sentence describing what it is
  • Responsibility — Exactly what it does (specific, not vague)
  • Instructions — Step-by-step approach, like onboarding a new team member
  • Tools — Which tools it's allowed to use
  • Output — What it produces when done

Example agents Weave might create

  • planner — Breaks down tasks into implementation steps
  • executor — Implements code following plans
  • reviewer — Reviews for quality, bugs, and conventions
  • tester — Writes and runs tests
  • security — Checks for vulnerabilities (for projects handling sensitive data)

The key: agents are customized to your stack. Your reviewer knows your ORM patterns. Your tester matches your testing framework. Nothing is generic.

Adding agents later

/weave:add-agent

This starts a short conversation about what the agent should do, then generates it and offers to add it to an existing team.

Teams

Teams are groups of agents that work together on complex tasks. Each team defines:

  • Purpose — What workflow this team handles
  • Members — Which agents participate
  • Workflow order — Who does what first, who hands off to whom
  • Rejection handling — What happens when the reviewer says no
  • Trigger — A skill/command that activates the team

Example: Feature Development Team

Trigger: /weave:build-feature

  1. Planner    → breaks the feature into steps
  2. Researcher → analyzes relevant codebase areas
  3. Executor   → implements the code
  4. Reviewer   → checks quality + conventions
     ↳ reject? → back to Executor with feedback
  5. Tester     → writes and runs tests

Adding teams later

/weave:add-team

Skills

Skills are custom slash commands that automate specific tasks. They're markdown files with instructions for the AI.

Weave comes with 7 built-in skills:

SkillWhat it does
/weave:onboardingStart the onboarding conversation
/weave:evolveOptimize workflow based on project changes
/weave:statusShow current workflow setup
/weave:syncSync workflow across platforms
/weave:add-agentCreate a new agent
/weave:add-skillCreate a new custom skill
/weave:add-teamDefine a new agent team

During onboarding, Weave creates additional project-specific skills based on your needs (e.g., /weave:deploy, /weave:review).

Evolve

Your project changes. Your workflow should too.

Weave has two mechanisms for this:

Background hooks

Two hooks run automatically:

  • evolve-hook — Tracks file edit patterns. When it detects significant changes (new frameworks, repeated tasks), it suggests running /weave:evolve
  • context-guard — Monitors context window usage and warns before it overflows

Manual evolution

/weave:evolve

The optimizer agent reads your git history, current codebase, and existing setup. It looks for:

  • New technologies that need specific rules
  • Stale agents that don't match the project anymore
  • Repeated patterns that could be a skill
  • Missing coverage (tests, CI, docs)

Every change is proposed and explained before being applied. Nothing happens silently.

Multi-Platform Sync

Weave uses a single source of truth (.weave/) and translates it into each platform's native format.

How it works

  • All definitions live in .weave/ (agents, skills, teams, rules)
  • The sync agent reads .weave/platforms.json to know which platforms are active
  • For each platform, it writes files in the correct native format
  • Each platform gets the maximum features it supports

Uninstall & Purge

Remove from project

$ npx weave-ai uninstall

Requires two confirmations ("yes" + type "UNWEAVE"). Removes every Weave file from the current project — skills, agents, hooks, rules, .weave/ directory. If CLAUDE.md or AGENTS.md has non-Weave content, only the Weave section is removed.

Remove from system

$ npx weave-ai purge

Requires three confirmations. Does everything uninstall does, plus:

  • Removes the global npm package
  • Clears the npx cache for weave-ai

After purge, it's as if Weave was never installed.
