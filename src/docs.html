Why Weave?

AI coding tools are powerful, but they don't know you. They don't know your project's architecture, your coding conventions, or how you like to break down tasks. So you end up repeating yourself — explaining context, correcting habits, re-establishing rules every session.

Weave fixes this. It's a framework that lives inside your AI tool and gives it persistent, personalized knowledge about:

  • Who you are — your role, experience, and how you like to work
  • What you're building — your tech stack, architecture, and project goals
  • How you want AI to behave — your rules, conventions, and preferences

Instead of configuring this manually, Weave generates everything through a conversation. And it keeps improving as your project evolves.

What makes Weave different

  • No templates — Everything is generated specifically for you and your project
  • Lives inside your tool — Not a separate CLI or service. It runs as native skills/commands within your AI editor
  • Conversation-based — The onboarding is a natural dialog, not a form
  • Auto-evolving — Hooks track your patterns and suggest improvements
  • Multi-platform — One source of truth, synced to all your AI tools natively
  • Zero lock-in — Uninstall removes everything cleanly

Installation

Weave installs via npx — no global packages needed:

$ npx weave-ai

The installer will:

  1. Show the Weave banner
  2. Auto-detect which AI tools are present in your project
  3. Let you choose which platforms to install for
  4. Install platform-native files for each selected platform
  5. Create the .weave/ state directory

Requirements

  • Node.js 18+
  • A project directory (Weave installs into the current directory)
  • At least one supported AI tool set up in the project

Onboarding

After installation, open your AI tool in the project and run:

/weave:onboarding

This starts a natural conversation (~10 exchanges) in three phases:

Phase 1 — The Person

Weave learns about you:

  • Your role and experience level
  • Your primary tech stack
  • How you prefer to work with AI (plan first vs. dive in, terse vs. detailed)
  • What frustrates you about your current workflow

Phase 2 — The Project

Weave analyzes your codebase (package.json, directory structure, frameworks) and asks about:

  • Your vision and goals for the project
  • The architecture and key decisions
  • Your team structure and collaboration style
  • Current pain points and priorities

Phase 3 — Workflow Proposal

Based on everything learned, Weave proposes a complete workflow:

  • Which agents to create and why
  • Which teams to assemble with what workflow
  • Which rules to enforce
  • Which custom skills to add

You review, adjust, and approve. Then Weave generates all the files.

Generated output

.weave/
├── config.json        # Your profile and project context
├── agents/*.md        # Agent definitions
├── teams/*.json       # Team configurations
├── rules/*.md         # Project rules
└── skills/*.md        # Custom skills

Agents

Agents are specialized AI personas with a single clear job. Each agent has:

  • Role — One sentence describing what it is
  • Responsibility — Exactly what it does (specific, not vague)
  • Instructions — Step-by-step approach
  • Tools — Which tools it's allowed to use
  • Output — What it produces when done

Example agents

AgentJob
plannerBreaks down tasks into implementation steps
executorImplements code following plans
reviewerReviews for quality, bugs, and conventions
testerWrites and runs tests
securityChecks for vulnerabilities

Every agent is customized to your stack. Your reviewer knows your ORM patterns. Your tester matches your testing framework. Nothing is generic.

Creating agents

/weave:add-agent

Starts a short conversation about what the agent should do, generates it, and offers to add it to an existing team.

Teams

Teams are groups of agents that work together on complex tasks. Each team defines:

  • Purpose — What workflow this team handles
  • Members — Which agents participate
  • Workflow order — Who does what first, who hands off to whom
  • Rejection handling — What happens when the reviewer says no
  • Trigger — A skill that activates the team

Example: Feature Development Team

Trigger: /weave:build-feature

  1. Planner    → breaks the feature into steps
  2. Researcher → analyzes relevant codebase areas
  3. Executor   → implements the code
  4. Reviewer   → checks quality + conventions
     ↳ reject? → back to Executor with feedback
  5. Tester     → writes and runs tests

Creating teams

/weave:add-team

Define purpose, pick agents, set workflow order, and Weave generates the team config plus a trigger skill.

Skills

Skills are custom slash commands that automate specific tasks. They're markdown files with instructions for the AI.

Built-in skills

SkillWhat it does
/weave:onboardingStart the onboarding conversation
/weave:evolveOptimize workflow based on project changes
/weave:statusShow current workflow setup
/weave:syncSync workflow across platforms
/weave:add-agentCreate a new agent

/weave:add-teamDefine a new agent team

During onboarding, Weave also creates project-specific skills based on your needs — e.g. /weave:deploy, /weave:review, /weave:refactor.

Creating skills

/weave:add-skill

Describe what the skill should do and Weave generates it.

Evolve

Your project changes. Your workflow should too.

Background hooks

Two hooks run automatically in the background:

  • evolve-hook — Tracks file edit patterns. When it detects significant changes (new frameworks, repeated tasks), it suggests running /weave:evolve
  • context-guard — Monitors context window usage and warns before it overflows

Manual evolution

/weave:evolve

The optimizer agent reads your git history, current codebase, and existing setup. It looks for:

  • New technologies that need specific rules
  • Stale agents that don't match the project anymore
  • Repeated patterns that could become a skill
  • Missing coverage (tests, CI, docs)

Every change is proposed and explained before being applied. Nothing happens silently.

Multi-Platform Sync

Weave uses a single source of truth (.weave/) and translates it into each platform's native format.

How it works

  1. All definitions live in .weave/ (agents, skills, teams, rules)
  2. The sync agent reads .weave/platforms.json to know which platforms are active
  3. For each platform, it writes files in the correct native format
  4. Each platform gets the maximum features it supports

Platform-native formats

ConceptClaude CodeOpenClawCursorCodex
Skills .claude/commands/ skills/sub-skills/ .cursor/commands/ .agents/skills/
Agents .claude/agents/ SKILL.md weave.mdc AGENTS.md
Rules CLAUDE.md SOUL.md weave.mdc AGENTS.md
Hooks .claude/hooks/ hooks/ .cursor/hooks/ .codex/hooks/ 
/weave:sync

Run this after making changes to propagate everything across platforms.

File Structure

After onboarding, your project looks like this:

project/
├── .weave/                    # Source of truth
│   ├── config.json            # User + project context
│   ├── platforms.json         # Active platforms
│   ├── agents/*.md            # Universal agent definitions
│   ├── skills/*.md            # Universal skills
│   ├── teams/*.json           # Team definitions
│   ├── rules/*.md             # Universal rules
│   └── history/               # Change log
│
├── .claude/                   # Claude Code (native)
│   ├── commands/weave:*.md    # Skills as slash commands
│   ├── agents/weave-*.md      # Subagents
│   └── hooks/                 # Lifecycle hooks
│
├── .cursor/                   # Cursor (native)
│   ├── commands/weave-*.md    # Commands
│   ├── rules/weave.mdc        # Rules
│   └── hooks.json + hooks/    # Lifecycle hooks
│
├── .agents/skills/            # Codex (native)
├── .codex/hooks.json          # Codex hooks
│
├── skills/weave/              # OpenClaw (native)
├── SOUL.md / STYLE.md         # OpenClaw identity
├── AGENTS.md                  # Codex/OpenClaw agents
└── CLAUDE.md                  # Claude Code rules

The .weave/ directory is the canonical source. Platform directories contain generated files that are kept in sync.

Uninstall & Purge

Remove from project

$ npx weave-ai uninstall

Requires two confirmations:

  1. Type yes to confirm
  2. Type UNWEAVE to finalize

This removes every Weave file from the current project — skills, agents, hooks, rules, and the entire .weave/ directory. If CLAUDE.md or AGENTS.md has non-Weave content, only the Weave section is removed.

Remove from system

$ npx weave-ai purge

Requires three confirmations. Does everything uninstall does, plus:

  • Removes the global npm package
  • Clears the npx cache for weave-ai

After purge, it's as if Weave was never installed.
